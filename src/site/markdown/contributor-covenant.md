# Contributor Covenant

## Information

The Contributor Covenant is a widely adopted code of conduct for open source communities. It was created by Coraline
Ada Ehmke in 2014 and defines expected behavior and enforcement responsibilities for project contributors and
maintainers.

Key versions:

* **1.x** — original format, widely used in projects from 2014 onward.
* **2.x** — revised with clearer enforcement guidelines, explicit enforcement ladder, and updated scope language.
  Version 2.1 is the current recommended version.

Projects adopt a code of conduct to:

* set clear expectations for contributor behavior,
* provide a safe and welcoming environment,
* give maintainers a documented basis for enforcement decisions.

### Adding to a Project

1. Copy the code of conduct text into `CODE_OF_CONDUCT.md` at the project root.
2. Fill in the contact email for reporting violations.
3. Link to `CODE_OF_CONDUCT.md` from the project `README.md`.
4. Reference it in `CONTRIBUTING.md` if one exists.

## Usage, tips and tricks

### Enforcement Ladder

The Contributor Covenant 2.x defines a four-step enforcement ladder for maintainers to follow:

1. **Correction** — private written warning, clarifying the violation and expected behavior.
2. **Warning** — formal warning with consequences defined for continued behavior. No interaction with involved parties
   for a specified period.
3. **Temporary Ban** — temporary removal from all project interaction (issues, PRs, community channels).
4. **Permanent Ban** — permanent removal from all project interaction for repeated or severe violations.

Maintainers should apply the ladder consistently and document decisions privately.

### Responsibilities

Project maintainers are responsible for:

* clarifying standards,
* taking appropriate action on reported violations,
* keeping reports confidential.
