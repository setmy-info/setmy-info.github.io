# Patents

## Information

`Patent Public Search` is the USPTO web application for searching published patents and patent application materials.

It is useful for quick prior art checks, patent lookups, and general patent research.

# Prior Art Search Guide – Home Reference

## 1. Purpose

The purpose of this guide is to help you independently determine whether your invention idea already exists (i.e.,
whether there is existing prior art).

This is an important step before writing a patent application because it:

- helps avoid duplicating existing inventions
- helps define what is unique about your invention
- saves time and money

## 2. Where to Search for Patents and Technical Solutions

### 2.1 Google Patents

https://patents.google.com

The easiest and most beginner-friendly tool.

It allows you to:

- search using keywords
- filter by country and publication date
- view patent descriptions and claims

### 2.2 Espacenet (European Patent Office)

https://worldwide.espacenet.com

Suitable for more comprehensive searches.

It allows you to:

- search over 150 million patent documents
- use CPC and IPC classification codes
- find patent families filed in different countries

### 2.3 USPTO (United States Patent and Trademark Office)

https://ppubs.uspto.gov

The official U.S. patent database.

Useful if:

- your invention may target the U.S. market
- you want to review official patent claims

## 3. Describe Your Invention Before Searching

Write a one- or two-sentence technical description.

Example:

"A system that optimizes food courier allocation in real time using machine learning."

Avoid descriptions such as:

- food app
- smart ordering

Instead, use technical language such as:

- system
- method
- algorithm
- optimization
- data processing

## 4. Finding Keywords (A Very Important Step)

Create a list of alternative keywords.

Example (food delivery):

- food delivery optimization
- courier routing system
- dynamic dispatch algorithm
- real-time logistics scheduling
- on-demand delivery system
- machine learning routing

## 5. Search Strategy

Never rely on a single search query.

Try multiple variations.

### Basic search

- food delivery system

### Technical search

- dynamic vehicle routing algorithm

### AI / data-oriented search

- machine learning logistics optimization

### Problem-oriented search

- reducing delivery latency system

## 6. How to Evaluate Search Results

For every patent, review:

### 6.1 Abstract

Provides a quick overview.

### 6.2 Claims (The Most Important Section)

Defines the actual scope of legal protection.

Always read at least Claim 1.

### 6.3 Drawings and Description

Helps you understand the technical implementation.

## 7. What to Look For (Checklist)

Determine whether an existing invention:

- performs the same function in the same way
- solves the same technical problem using a similar solution
- uses the same type of algorithm or architecture
- optimizes the same process (for example routing, prediction, or scheduling)

## 8. Patent Classification Codes (Advanced)

For more precise searching, use CPC classification codes.

Examples:

- G06Q — Business methods and commercial systems
- G06N — Artificial intelligence and machine learning
- G08G — Traffic control and routing systems

## 9. Evaluating Similar Patents

If you find a similar patent, ask yourself:

- Is this exactly the same solution?
- Does my invention differ technically?
- Is the difference only business logic? (Usually a weak distinction.)
- Is the difference in the technical implementation? (Usually a stronger distinction.)

## 10. Important Limitation

An independent prior art search:

- is not legal advice
- does not guarantee patentability
- does not replace a professional patent attorney's analysis

However, it is an excellent first step.

## 11. Summary

A good prior art search includes:

- multiple keyword variations
- multiple patent databases
- careful review of the claims
- evaluation of technical similarities

If no similar inventions are found,
there is a greater possibility that your invention is novel.

If many similar inventions exist,
your invention may need to be refined with a more distinctive technical implementation.

## 12. Final Thought

The best patent idea is not necessarily a completely new idea.

Instead, it is frequent:

**A technically better solution to an existing problem.**
